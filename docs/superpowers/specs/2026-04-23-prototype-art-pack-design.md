# 恋爱地图对话游戏首包美术素材设计文档

## 1. 概述

本设计文档定义当前项目第一轮美术接入方案，目标是在不改动核心玩法结构的前提下，为现有原型补上一套“最低可玩素材包”。

这套素材包不追求一次性覆盖全部 16 个子场景，而是优先解决当前项目最缺失的三类视觉资产：

- 世界地图
- 区域主背景
- 角色半身立绘

完成后，项目应从“纯功能原型”提升为“可展示、可试玩、气质统一的恋爱游戏原型”。

## 2. 设计目标

- 在手机竖屏布局下，为上半屏提供稳定、统一的主视觉内容
- 保持当前“地图探索 -> 场景进入 -> 剧情事件 -> 自然语言对话”的结构不变
- 通过最少数量的素材，显著改善沉浸感和正式感
- 为后续补充子场景背景、美术变体、角色多表情立绘预留清晰扩展路径
- 将素材生成、命名、存放和运行时映射方式标准化，避免后续返工

## 3. 非目标

本轮不包含以下内容：

- 16 个子场景全部独立背景图
- 角色多套服装、多表情、多姿态立绘
- 动画演出、Live2D、骨骼动画或镜头特效
- 图标库、复杂 HUD、完整 UI 重绘
- 音效、配乐、转场音频
- 剧情 CG 或关键事件插图

## 4. 视觉方向

### 4.1 核心风格

首包素材统一采用以下方向：

- 偏二次元手游立绘
- 现代校园恋爱视觉小说气质
- 画面干净、柔和、带轻微梦幻感
- 以傍晚、夜色、室内暖光为主要氛围来源
- 更强调情绪停顿和留白，而不是高强度戏剧冲突

### 4.2 视觉关键词

所有素材在生成时应尽量复用同一组描述关键词，以减少风格漂移：

- 日系恋爱视觉小说
- 现代校园
- 轻柔光影
- 干净轮廓
- 粉紫蓝夜色氛围
- 手机竖屏构图友好
- 细腻、克制、暧昧

### 4.3 统一性原则

- 地图、背景、立绘必须看起来像同一部作品中的素材，而不是不同模型批次的拼贴
- 角色立绘颜色不能明显脱离背景主色调
- 背景图要给对白区域与角色叠放留出安全空间
- 地图与背景都要优先服务“地点识别”和“气氛建立”，不追求复杂细节堆砌

## 5. 第一轮素材范围

### 5.1 必做素材

首轮素材包包含以下内容：

1. 城市总地图 1 张
2. 区域主背景 4 张
3. 角色半身立绘 2 张
4. 通用 UI 装饰素材 2 到 4 张

### 5.2 素材清单

#### 地图

- 城市总地图
  - 用于游戏初始世界地图阶段
  - 需能承接学校、医院、商场、主角家四个热点区域

#### 区域主背景

- 学校主背景
- 医院主背景
- 商场主背景
- 主角家主背景

这些背景不直接等于某一个子场景，而是对应每个大区域的代表性氛围图，用于在第一轮替代纯色或占位背景。

#### 角色立绘

- 林澄半身立绘
- 周然半身立绘

第一轮每位角色仅做一张通用半身立绘，用于事件中展示角色存在感。

#### UI 装饰素材

- 对话区域柔和纹理底图
- 地点标题装饰牌
- 按钮或选项轻装饰底纹
- 事件层柔光叠加图

UI 装饰素材最终数量可根据生成质量控制在 2 到 4 张。

## 6. 素材用途与运行时映射

### 6.1 地图素材用途

城市总地图替换当前 Phaser 场景中的纯色底与简陋点位视觉，保留现有四个区域逻辑和热点分布。第一轮中，地图仍然承担“区域入口选择”职责，不承载复杂路线或二级地图信息。

### 6.2 背景素材用途

区域主背景作为上半屏主视觉，在以下场景下使用：

- 玩家进入某个区域但尚未进入事件时
- 玩家停留在该区域的任一子场景时
- 玩家结束该区域内事件后返回探索态时

第一轮不区分“教室”和“走廊”这种细粒度背景，只做区域级复用。

### 6.3 立绘素材用途

角色半身立绘主要用于事件阶段：

- 当事件中存在固定角色时，在上半屏背景上叠加角色半身图
- 没有固定角色时，仅展示背景，不强行出现角色图
- 角色图优先支持一名主角色显示，避免第一轮就处理双人同屏构图

### 6.4 UI 素材用途

UI 装饰素材仅用于增强气质，不改变当前交互结构：

- 给状态条、地点标题或故事框增加轻装饰
- 优化按钮与面板的视觉层次
- 保持当前 DOM 结构基本稳定

## 7. 资源命名与目录方案

### 7.1 存放位置

第一轮素材建议统一放在 `public/assets` 下，便于 Vite 直接访问。

建议目录结构如下：

- `public/assets/map`
- `public/assets/backgrounds`
- `public/assets/characters`
- `public/assets/ui`

### 7.2 命名规范

建议使用稳定、可扩展的英文 snake-case 或 kebab-case 命名：

- `public/assets/map/city-overview-main.png`
- `public/assets/backgrounds/region-school-main.png`
- `public/assets/backgrounds/region-hospital-main.png`
- `public/assets/backgrounds/region-mall-main.png`
- `public/assets/backgrounds/region-home-main.png`
- `public/assets/characters/lin-cheng-half-body.png`
- `public/assets/characters/zhou-ran-half-body.png`
- `public/assets/ui/dialogue-panel-texture.png`
- `public/assets/ui/location-title-plate.png`

### 7.3 扩展规则

后续如果补充子场景图，可沿用同一命名体系：

- `scene-classroom-main.png`
- `scene-cafe-main.png`
- `scene-balcony-main.png`

后续如果补充角色变体，可继续追加后缀：

- `lin-cheng-half-body-smile.png`
- `lin-cheng-half-body-night.png`

## 8. 接入策略

### 8.1 总体策略

第一轮接入遵循“低侵入替换视觉层”的原则：

- 不重写玩法状态机
- 不改变现有探索、事件、记忆、聊天逻辑
- 优先替换上半屏与事件层视觉呈现
- 在必要处增加极少量状态选择逻辑，用于决定当前显示哪张图

### 8.2 地图接入

地图层需要把当前 Phaser 占位视觉替换为正式地图底图，同时保留现有可点击区域逻辑。第一轮允许继续用简单热点定位，不要求立刻做成复杂的精确碰撞区域。

### 8.3 背景接入

当前 UI 需要新增“按当前位置解析背景图”的能力：

- 未进入任何区域时显示城市总地图
- 进入区域后显示该区域主背景
- 进入事件时优先显示当前事件所在区域背景

### 8.4 角色接入

当事件中存在主要出场角色且该角色已有立绘资源时，在视觉区叠加对应半身图。第一轮以单角色展示为主，位置应避免遮挡主要文本区域。

### 8.5 UI 接入

UI 装饰素材以 CSS 背景图或轻量层叠图片形式加入，避免引入额外复杂渲染机制。

## 9. 生成策略

### 9.1 生成顺序

建议按以下顺序生成和确认素材：

1. 角色设定图
2. 城市总地图
3. 四张区域主背景
4. UI 装饰素材

这个顺序的原因是：

- 角色立绘会定义人物气质与作品完成度
- 地图会定义整体世界观观感
- 背景图应跟随角色和地图的色彩语言
- UI 装饰应最后贴合整体画面风格

### 9.2 角色生成原则

- 统一面部完成度、线稿风格、服装复杂度
- 优先做适合对话场景的半身构图
- 透明背景优先，便于直接叠加到场景背景之上
- 林澄需要明显具备“核心女主”的辨识度
- 周然要有辅助角色气质，但不能抢主角视觉中心

### 9.3 背景生成原则

- 更重氛围与留白，不做过度拥挤细节
- 画面中预留角色叠放区域
- 颜色上与角色立绘兼容
- 学校和商场承担更强剧情推进感
- 医院和主角家承担情绪延展与私密感

### 9.4 地图生成原则

- 更像恋爱冒险游戏区域选择图，而不是现实导航地图
- 四个主区域需要一眼可辨
- 构图需要适配手机竖屏上半屏
- 要允许后续继续在图上精修热点位置

## 10. 风险与对应策略

### 10.1 风格漂移

风险：不同批次图片可能像不同项目。

应对：

- 固定一套核心关键词
- 先确定角色，再让地图和背景跟随角色风格
- 对不统一的图片优先重生成，而不是硬接入

### 10.2 图量过大导致首轮失控

风险：如果一开始就补满全部场景图，返工成本高。

应对：

- 首轮只做区域主背景
- 子场景图延后到第二轮

### 10.3 立绘与 UI 打架

风险：角色图可能压住地图、地点标题或重要文本区域。

应对：

- 第一轮只在上半屏视觉区展示角色
- 下半屏对话区保持原有结构
- 立绘默认靠边放置，保留中间识别空间

### 10.4 资源接入后结构变乱

风险：后续补图时命名和映射失控。

应对：

- 先定义目录与命名规范
- 把资源映射逻辑集中在单独的视觉资源模块里

## 11. 验收标准

首轮素材接入完成后，应满足以下结果：

- 游戏启动后，世界地图阶段可看到正式城市地图
- 进入不同区域时，上半屏能切换到对应主背景
- 进入有固定角色的事件时，能显示对应半身立绘
- 整体视觉风格统一，不再呈现纯占位原型感
- 现有测试与构建不因素材接入而失效

## 12. 后续扩展路径

本设计文档完成的是第一轮“最低可玩素材包”。在此基础上，第二轮可自然扩展为：

- 每个子场景独立背景图
- 角色多表情、多姿态立绘
- 关键剧情 CG
- 更完整的 UI 视觉升级
- 特殊时间段背景变体

第一轮的实现必须保证这些扩展可以在不推翻目录结构与映射规则的前提下继续进行。
